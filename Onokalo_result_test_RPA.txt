Аналіз процесу
	1.Вхідні дані:
		* Конфіг - шлях до папки з файлами, дані для логіну на веб порталі
		* Файли CSV у зазначеній папці і їх зміст (поля Product_Code, Client_ID,  ім'я клієнта, кількість товару, найменування товару, вартість товару)
		* І сам веб портал що повертає значення (Good / Bad / Unknown) по Client_ID	
		
	  Вихідні дані:
		* Внесенні в CRM дані про замовлення
		* Файл Daily_Report.xlsx
		* ФАйл з логами помилок
		* Оброблені файли у папці "\опрацьовані\"
		
	2. Бізнес правила:
		* ЯКЩО в полі Product_Code є 'PROMO_001' то це замовлиння вважається "пріоритетним"
		* КОЛИ Веб портал повертає (Good / Bad / Unknown) записати це значення
		* ЯКЩО при виконанні виникає помилка, внести це в "лозі помилок" і перейти до наступного файлу
		* ПІСЛЯ успішної обробки файл переміщується в ПІДпапку "\опрацьовані\". 
			/* В завданні не визначено явно, що робити з файлами після обробки. Але для уникнення повторної обробки пропоную їх переміщати.*/

Алгоритмічна логіка
	* Ініціалізація
		1. Зчитати конфігурацію (шлях до папки, дані логіну)
		2. Відкрити/створити "лог помилок"
		3. Перевірити існування папки з замовленнями
			ЯКЩО папки не існує, зробити запис в "лозі помилок" і завершити роботу
		4.Отримати список всіх файлів CSV
			ЯКЩО їх не існує, зробити запис в "лозі помилок" і завершити роботу
		5. Створити підпапку \опрацьовані\
		6. Відкрити/створити Daily_Report.xlsx
			ЯКЩО файл створюється вперше, створити заголовки колонок:
				| ProcessingDate |
				| Client_ID |
				| ClientName |
				| ProductName |
				| Quantity |
				| Price |
				| Credit_Status |
				| Is_Priority |
				| Processing_Result |		
				
		7. Відкрити веб портал (orders.client.com)
			знайти поле логіну
			ввести логін
			знайти поле паролю
			ввести пароль
			знайти кнопку входу
			натиснути
			перевірити успішність входу
				ЯКЩО не вдалося - повторити 3 рази з очікуванням 30 секунд
					ЯКЩО всі спроби невдалі:
							зробити запис в "лог помилок"
							(відправити повідомлення відповідальному працівнику)
							завершити роботу

		8. Відкрити CRM
	* Головний цикл (Loop)
		ДЛЯ КОЖНОГО файлу в папці:
			1. ВІдкрити як таблицю
			2. Пересвідчитись в наявності обовязкових полів (поля Product_Code, і т.д.)				
				ЯКЩО помилка зробити запис в "лозі помилок" і перейти до наступного файлу
			
			/* В завданні явно не вказано кількість замовлень у файлі, 
			тому припускаємо що файл може містити декілька рядків */
			
			3. ДЛЯ КОЖНОГО рядка зчитати всі обовязкові поля (сформувати колекцію замовлень)
				ЯКЩО Product_Code = 'PROMO_001', Is_Priority = True
				ІНАКШЕ Is_Priority = False
				ЯКЩО помилка читання рядка
					зробити запис в "лог помилок"
					перейти до наступного рядка
			4. Переключитись на веб портал
			5. ДЛЯ КОЖНОГО замовлення з колекції 
				взяти Client_ID
				знайти поле вводу
				внести значення Client_ID
				знайти кнопку пошуку
				натиснути кнопку пошуку
				зчитати значення:	Good / Bad / Unknown
				записати його як credit_status
					ЯКЩО Client_ID не знайдено
						повторити 3 рази з очікуванням 30 секунд
						ЯКЩО всі спроби невдалі:
							зробити запис в "лог помилок"
							перейти до наступного замовлення
					ЯКЩО отримано 401 Unauthorized
						або редірект на сторінку логіну:
						автоматично виконати повторну авторизацію
					ЯКЩО веб портал недоступний:
						повторити 3 рази з очікуванням 30 секунд
							ЯКЩО всі спроби невдалі:
								зробити запис в "лог помилок"
								зробити screenshot помилки
								перейти до наступного файлу
			6. Перемикання на CRM
				знайти поле ClientName
				внести значення
				знайти поле Quantity
				внести значення
				знайти поле ProductName
				внести значення
				знайти поле Price
				внести значення
				знайти поле CreditStatus
				внести значення
				знайти поле Is_Priority
				внести значення
				натиснути кнопку Save
				перевірити повідомлення про успішне збереження
					ЯКЩО запис не створено:
						зробити запис в "лог помилок"
						перейти до наступного замовлення

			7. ПІСЛЯ завершення обробки всіх рядків файлу:
				Для кожного замовлення додати рядок у Daily_Report.xlsx

				перемістити файл в "\опрацьовані\"

Стійкість та Покращення
	1. Веб портал недоступний - пропоную реалізувати механізм повторних спроб (3 по 30с) і опційно відправка повідомлення відповідальній особі.
	2. Некоректний або пошкоджений CSV-файл - пропоную перевіряти наявність обов'язкових полів і наявність підходящих значень в цих полях, при помилці - запис у лог помилок і перехід до іншого файлу. Пошкоджений файл не переносится в опрацьовані, очікує ручної обробки.
	3. Помилка авторизації на порталі (сесія закінчилась) - пропоную після 401 Unauthorized або редірект на сторінку логіну - автоматично повторювати авторизацію.
	
Масштабування та оптимізація:
	1. Багатоканальне джерело вхідних даних
	Зараз робот читає лише CSV-файли з папки. В майбутньому можна розширити джерела:
		* Email-інтеграція: бот моніторить поштову скриньку і автоматично зберігає вкладення-замовлення в папку (або обробляє напряму).
		* API-інтеграція: замість CSV-файлів замовлення надходять через REST API інтернет-магазину напряму в робота, що усуває ризик помилок файлової системи.
		* SharePoint / Google Drive: моніторинг хмарної папки замість локальної — для розподіленої команди.
	2. Оркестрація та паралельна обробка
	Якщо кількість замовлень зросте (десятки/сотні CSV на день):
		* Впровадити Orchestrator для планування запусків, моніторингу та повторних запусків при збоях.
		* Розбити процес на кілька паралельних роботів: один читає CSV і формує чергу (Queue), інші незалежно обробляють кожен елемент черги й звертаються до порталу — це значно прискорить обробку.
		* Додати дашборд з метриками: скільки оброблено, скільки помилок, середній час обробки одного замовлення.
